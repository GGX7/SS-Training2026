メモリの概念

所要時間 約 90 分
前提 Day 01, 02 完了
使うもの VS Code + Python 3.11
// TODAY'S GOAL
Pythonにおけるデータ型（int, float, str, list, dict）を使い分けられるようになり、 「変数は箱ではなく名札である」という考え方を自分の言葉で説明できる状態になる。

なぜこれを学ぶのか

強化学習のコードを読み書きしていると、「同じリストを2箇所で変更したら片方しか変わらない（あるいは両方変わってしまう）」という現象に必ず出会います。 これは変数が「値をコピーした箱」ではなく「メモリ上のオブジェクトを指し示す名札」だからです。 この直感は、後々 NumPy の配列や、RLlib の config 辞書を扱うときに致命的なバグを防いでくれます。

01 · 基本のデータ型

Python には最初に覚えるべきデータ型が5つあります。数値（整数と小数）、文字列、リスト、辞書です。 ターミナルで python と打って対話モードを起動し、以下を実際に入力してみてください。

python · 対話モード 
# 整数 (int)
type(42)          # <class 'int'>

# 浮動小数 (float)
type(3.14)        # <class 'float'>

# 文字列 (str)
type("hello")     # <class 'str'>

# リスト (list) — 順序あり、変更可能
type([1, 2, 3])    # <class 'list'>

# 辞書 (dict) — キーと値のペア
type({"lr": 0.001}) # <class 'dict'>
// NOTE type() は「このオブジェクトの設計図は何？」を教えてくれる組み込み関数です。 これは今後クラスを学ぶ時に伏線として効いてきます。

02 · 変数は「名札」である

多くの入門書は「変数とは値を入れる箱」と説明しますが、この例えは Python においては間違いです。 正確には、変数はメモリ上のオブジェクトに貼り付けた名札です。次の例を見てください。

python · 名札の挙動 
a = [1, 2, 3]
b = a          # b は a と「同じリスト」への名札

b.append(4)

print(a)       # → [1, 2, 3, 4]  ← a も変わっている!
print(id(a) == id(b))  # → True

id() はオブジェクトのメモリ上の住所を返します。a と b は同じ住所を指しているので、 片方を変更するともう片方からも変わって見えます。これが「名札」の意味です。

// 詰まりポイント 設定辞書 config を別の関数に渡してその中で書き換えると、元の config も変わってしまうことがあります。 これを防ぐには import copy; copy.deepcopy(config) を使います。RLlibでは頻出のパターンなので覚えておきましょう。

03 · 手を動かす

以下の演習を実際に入力して、想定出力と一致するか確認してください。

演習 3-1 · 辞書を使ったハイパーパラメータの表現

python · exercise.py 
# 強化学習でよく使う設定の書き方を練習する
config = {
    "learning_rate": 0.001,
    "gamma": 0.99,
    "num_workers": 4,
}

# 値を取り出す
print(config["learning_rate"])  # → 0.001

# 新しいキーを追加
config["batch_size"] = 64
print(config)
// HINT 第4ヶ月で触れるRLlibの config は、まさにこの形の辞書です。今のうちに慣れておくと後が楽になります。

今日のまとめ

今日は Python の基本的なデータ型と、変数が「名札」であるという概念を扱いました。 この名札モデルは、来週学ぶリスト内包表記や関数への引数渡し、そして第2ヶ月のクラスの理解にも直結します。 違和感があっても一度は手を動かして、id() で住所を確認する癖をつけておいてください。
